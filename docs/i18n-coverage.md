## Reading the Columns

- **Translated** - keys whose value is different from the EN message.
- **Intentional identical** - keys whose EN value is an exact reviewed brand or technical term that should stay unchanged.
- **Placeholder identical** - keys that still ship byte-identical to EN and need translation review unless intentionally added to the reviewed list.
- **Missing** - keys absent from the locale file. `chrome.i18n.getMessage` will fall through to EN for these.
- **Coverage** - same as the Translated percentage; restated as the headline number.

## How to Improve Coverage

1. Open `scripts/generate-locales.js` and find the locale's `T['<locale>']` translation table.
2. Add entries mapping English messages to translated messages. Preserve the exact reviewed brand and technical terms above.
3. Re-run `node scripts/generate-locales.js` to rewrite `extension/_locales/<locale>/messages.json`.
4. Re-run `node scripts/i18n-coverage.js` to refresh this report and review the feature-copy queue.
5. For zh_CN, the locale is hand-maintained outside the generator; edit `extension/_locales/zh_CN/messages.json` directly.
